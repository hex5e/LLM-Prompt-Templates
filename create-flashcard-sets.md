We will build flashcard sets about [X], one subtopic at a time.

---

Workflow:

1. First, list 5–8 subtopics you plan to cover for [X]. Ask me to confirm, reorder, add, or remove subtopics before continuing.
2. Once I confirm, begin with subtopic 1. Generate 5–10 flashcards numbered {SetNum}.{CardNum} (e.g. 1.1, 1.2, …).
3. After each batch, ask: "More for this set, or continue to the next?"
   - If I say **more** → generate the next 5–10 cards for the same subtopic, continuing the numbering (e.g. 1.11, 1.12, …).
   - If I say **continue** → move to the next subtopic, restart card numbering (e.g. 2.1, 2.2, …).
   - If I name a specific subtopic → jump to that one.
4. Repeat until I say we are done.

---

A flashcard has a Front (question) and a Back (answer).

Flashcard requirements:
- Atomic: Front/question must test exactly one fact, distinction, mapping, role, or mechanism.
- Clear: Front/question has one unambiguous answer.
- Least-context: Front/question provides the least context needed while still fulfilling the 'Clear' requirement.
- Recall-based: Front/question is answerable from memory, not just recognition.
- Compact: Back/answer is short, should usually be a couple words, and only rarely one short sentence.
- Meaningful: worth remembering.

Prefer:
- distinctions
- roles
- mappings
- cause/effect

Avoid:
- broad "What is X?" cards
- paragraph-length backs
- multi-part answers
- cards that list several properties at once
- vague or subjective wording
- binary questions true/false, either/or, this or that, etc
- If a concept is too broad, split it into multiple smaller cards.
- Do not include explanations.

Before outputting, check every card against:
- atomic
- clear
- least-context
- recall-based
- compact
- meaningful

Rewrite any card that fails.
